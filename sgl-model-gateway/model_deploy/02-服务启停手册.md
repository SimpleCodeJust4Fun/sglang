# SGLang 服务启停与调试手册

## 🚀 快速启动

### Windows PowerShell 启动
```powershell

```

### WSL 内部启动
```bash
cd ~
bash /mnt/e/dev/model-deploy/start_sglang.sh
```

## ⏸️ 服务停止

### 方法 1: 键盘中断（推荐）
如果服务在前台运行，按 `Ctrl+C` 即可优雅停止。

### 方法 2: 查找并终止进程
```bash
# 在 WSL 中执行
# 查找 SGLang 进程
ps aux | grep sglang

# 终止进程（替换 <PID> 为实际进程号）
kill <PID>

# 强制终止（如果正常终止无效）
kill -9 <PID>
```

### 方法 3: 终止所有 Python 进程（谨慎使用）
```bash
pkill -f "sglang.launch_server"
```

## 🔄 服务重启

```bash
# 停止服务
pkill -f "sglang.launch_server"

# 等待 2 秒确保进程完全退出
sleep 2

# 启动服务
bash /mnt/e/dev/model-deploy/start_sglang.sh
```

## 🔧 常见问题排查

### 问题 1: 端口已被占用
**症状**: 启动时报错 `Address already in use`

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :30000

# 终止该进程
kill <PID>
```

### 问题 2: 显存不足 (OOM)
**症状**: 启动时报错 `CUDA out of memory`

**解决方案**:
编辑 `start_sglang.sh`，降低 `--mem-fraction-static` 参数：
```bash
--mem-fraction-static 0.6  # 从 0.7 改为 0.6
```

### 问题 3: 模型加载失败
**症状**: 提示找不到模型文件

**解决方案**:
```bash
# 检查模型是否存在
ls -lh ~/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ/

# 重新下载模型
~/qwen_env/bin/python3 /mnt/e/dev/model-deploy/download_awq.py
```

### 问题 4: CUDA 环境错误
**症状**: 提示 `nvcc not found` 或 `CUDA_HOME not set`

**解决方案**:
```bash
# 检查 CUDA 是否安装
/usr/local/cuda/bin/nvcc --version

# 如果未安装，需要重新安装 CUDA Toolkit
sudo apt-get install -y cuda-toolkit-12-8
```

### 问题 5: 服务启动后无响应
**症状**: 服务启动但 API 调用超时

**解决方案**:
```bash
# 1. 检查防火墙
sudo ufw status

# 2. 检查服务日志是否有报错

# 3. 尝试使用本地回环地址
curl http://127.0.0.1:30000/model_info

# 4. 重启服务
pkill -f sglang
bash /mnt/e/dev/model-deploy/start_sglang.sh
```

## ⚡ 性能优化建议

### 1. 启用 CUDA Graph（如果显存充足）
编辑 `start_sglang.sh`，移除 `--disable-cuda-graph` 参数：
```bash
~/qwen_env/bin/python3 -m sglang.launch_server \
  --model-path /home/tyliu/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ \
  --port 30000 \
  --host 0.0.0.0 \
  --mem-fraction-static 0.7
  # 移除了 --disable-cuda-graph
```

### 2. 调整批处理大小
```bash
--max-running-requests 4  # 增加并发请求数
--chunked-prefill-size 4096  # 增加预填充大小
```

### 3. 调整显存分配策略
```bash
--mem-fraction-static 0.8  # 如果显存充足，可以增加到 0.8
```

## 🔄 开机自启动（高级）

### 方法 1: 创建 systemd 服务（推荐）
```bash
# 创建服务文件
sudo nano /etc/systemd/system/sglang.service
```

服务文件内容：
```ini
[Unit]
Description=SGLang Qwen2.5-7B Service
After=network.target

[Service]
Type=simple
User=tyliu
WorkingDirectory=/home/tyliu
Environment="CUDA_HOME=/usr/local/cuda"
Environment="PATH=/usr/local/cuda/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
Environment="LD_LIBRARY_PATH=/usr/local/cuda/lib64"
ExecStart=/home/tyliu/qwen_env/bin/python3 -m sglang.launch_server --model-path /home/tyliu/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ --port 30000 --host 0.0.0.0 --mem-fraction-static 0.7 --disable-cuda-graph
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable sglang
sudo systemctl start sglang

# 查看状态
sudo systemctl status sglang

# 查看日志
sudo journalctl -u sglang -f
```

```bash
# 使用其他参数
python3 model_deploy/pd-test.py "Tell me a joke" --model qwen2.5-0.5b-instruct --max-tokens 100
```

### 高级选项

```bash
# 显示详细信息（包含 bootstrap room 映射、完整输入输出文本等）
```

### 日志关联说明

工具会自动通过以下方式关联日志：

1. **Request ID 匹配**: 通过 Gateway 返回的 `X-Request-ID` 头查找对应日志
2. **Response ID 匹配**: Decode 节点的 rid 通常与 API 响应中的 ID 一致
3. **Bootstrap Room 映射**: Prefill 和 Decode 日志中的 `bootstrap_room` 字段相同，表明它们处理的是同一个请求
4. **时间窗口关联**: 如果 ID 匹配失败，会使用时间窗口（默认 60 秒）进行匹配

### 关键字段说明

| 字段 | 说明 |
|------|------|
| `request_id` | Gateway 生成的请求标识 |
| `rid` | SGLang 节点内部请求标识 |
| `bootstrap_room` | Prefill→Decode KV Cache 传输的配对标识 |
| `bootstrap_host/port` | Prefill 发送 KV Cache 到 Decode 的目标地址 |
| `cached_tokens` | KV Cache 复用的 token 数量 |
| `e2e_latency` | 从接收到完成的端到端延迟 |

### 日志文件位置

- **Gateway 日志**: `/tmp/sgl-gateway-test.log`
- **Prefill 日志**: `/tmp/sglang-prefill-1.log`, `/tmp/sglang-prefill-2.log`
- **Decode 日志**: `/tmp/sglang-decode-1.log`, `/tmp/sglang-decode-2.log`

工具会自动发现这些日志文件，也可以通过 `--gateway-log`、`--prefill-log`、`--decode-log` 手动指定。
