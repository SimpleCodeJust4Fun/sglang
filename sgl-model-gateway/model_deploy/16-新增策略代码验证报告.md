## 已发现并修复的问题

### 1. Worker API 使用错误 ✅ 已修复

**问题描述**:
最初代码中错误地将 `priority()` 和 `cost()` 当作返回 `Option` 的方法调用：
```rust
// 错误代码
let priority = w.priority().unwrap_or(0);
let cost = w.cost().unwrap_or(1.0);
```

**实际情况**:
查看 `src/core/worker.rs:259-274` 发现这些方法直接返回值：
```rust
fn priority(&self) -> u32 { ... }  // 返回 u32，不是 Option<u32>
fn cost(&self) -> f32 { ... }      // 返回 f32，不是 Option<f32>
```

**修复方案**:
```rust
// 正确代码
let priority = w.priority();
let cost = w.cost();
```

**影响文件**:
- ✅ `src/policies/request_size_bucket.rs` - 已修复
- ✅ `src/policies/performance_aware.rs` - 已修复
- ✅ `src/policies/request_classification.rs` - 已修复

### 2. BasicWorkerBuilder API 使用错误 ✅ 已修复

**问题描述**:
测试代码中错误地使用了不存在的 builder 方法：
```rust
// 错误代码 - BasicWorkerBuilder 没有这些方法
BasicWorkerBuilder::new("http://w1:8000")
    .priority(10)
    .cost(1.0)
    .build()
```

**实际情况**:
查看 `src/core/worker_builder.rs` 发现 builder 没有 `.priority()` 和 `.cost()` 方法。这些值通过 labels 设置。

**修复方案**:
```rust
// 正确代码 - 使用 label 方法
BasicWorkerBuilder::new("http://w1:8000")
    .label("priority", "10")
    .label("cost", "1.0")
    .build()
```

**影响文件**:
- ✅ `src/policies/request_size_bucket.rs` - 测试代码已修复
- ✅ `src/policies/performance_aware.rs` - 测试代码已修复
- ✅ `src/policies/request_classification.rs` - 测试代码已修复
