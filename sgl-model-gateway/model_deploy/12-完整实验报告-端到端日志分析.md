## 第五部分：关键发现总结

### 5.1 PD分离架构验证

| 验证项 | 结果 | 证据 |
|--------|------|------|
| Gateway同时发送请求到P和D | ✓ | Gateway日志显示dual dispatch |
| Prefill计算KV cache | ✓ | Prefill-2日志: #new-token: 31 |
| Bootstrap传输KV cache | ✓ | Prefill-2日志: Sending KV cache via bootstrap |
| **Decode直接返回响应** | ✓ | Decode-2日志: Returning response to Gateway |
| 响应不经过Prefill | ✓ | Prefill-2在传输后即返回200 |

### 5.2 性能分析

| 阶段 | 耗时 | 占比 |
|------|------|------|
| Prefill (KV计算+传输) | ~45ms | 9.6% |
| Decode (token生成) | ~365ms | 78.0% |
| Gateway开销 | ~21ms | 4.5% |
| 网络传输 | ~38ms | 8.1% |
| **总计** | **~469ms** | **100%** |

**关键发现**:
1. **Decode是主要耗时阶段** (78%)
   - 自回归生成，每个token ~35ms
   - 10个token = 350ms

2. **Prefill相对快速** (9.6%)
   - 31个token的并行计算
   - Bootstrap传输108KB很快

3. **Bootstrap传输效率高**
   - 108KB数据传输 ~10ms
   - 使用mooncake传输引擎

### 5.3 显存使用

| 组件 | 可用显存 | KV Cache | 特点 |
|------|---------|----------|------|
| Prefill-1/2 | 11.69 GB | 1.22 GB | 剩余显存多 |
