## 重要发现

### 显存限制

**16GB显存无法同时运行两个7B-Instruct-AWQ实例：**

- Prefill Server (mem-fraction=0.35): 启动成功，占用 ~10GB
- Decode Server (mem-fraction=0.2-0.3): **全部OOM (退出码137)**

**原因分析：**
- 7B-AWQ模型权重约5.4GB
- 两个进程都能看到16GB总显存，但物理显存是共享的
- Prefill占用10GB后，剩余6GB不足以让Decode分配KV cache池
- 即使降低mem-fraction到0.2，SGLang内部的内存池计算仍然失败

**解决方案：**
- 使用更小的模型（1.5B-AWQ）成功实现PD分离
- 1.5B-AWQ模型权重约1.6GB，两个实例总占用约14GB

## 故障排查记录

### 问题1: Decode Server OOM
**症状**: 进程被kill，退出码137
**原因**: 16GB显存不足
**解决**: 改用1.5B模型

### 问题2: 后台进程启动失败
**症状**: nohup/setsid启动后进程立即退出
**原因**: pkill命令在脚本中杀死了自身
**解决**: 使用setsid + < /dev/null 确保进程独立运行

### 问题3: mem-fraction-static参数不够
**症状**: RuntimeError: Not enough memory
**原因**: 7B模型权重5.4GB + KV cache需要大量显存
**解决**: 降低模型大小或使用更高显存显卡
