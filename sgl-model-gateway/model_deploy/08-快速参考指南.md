# Multi-PD 快速参考指南

## 快速启动 (三步)

```bash
cd /mnt/e/dev/sglang/sgl-model-gateway/model_deploy
```

## 架构信息

### 服务端口
| 服务 | 端口 | 用途 |
|------|------|------|
| Prefill-1 | 30000 | 预处理请求，Bootstrap: 9000 |
| Prefill-2 | 30001 | 预处理请求，Bootstrap: 9001 |
| Decode-1 | 30010 | 生成响应 |
| Decode-2 | 30011 | 生成响应 |
| Gateway | 3000 | 统一入口，路由请求 |

### 模型
- **名称**: Qwen2.5-0.5B-Instruct
- **大小**: ~1GB (FP16)
- **路径**: `~/.cache/modelscope/hub/models/qwen/Qwen2___5-0___5B-Instruct`

### 显存使用
- **总使用**: ~15.5GB / 16GB
- **每个实例**: ~3.5-4GB

## 调度策略说明

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| round_robin | 轮流分配请求 | 负载均衡 |
| cache_aware | 优先使用有缓存的节点 | 减少KV传输 |
| random | 随机分配 | 简单测试 |

## 常见问题

**Q: 某个服务启动失败?**
```bash
# 检查日志
tail -50 /tmp/sglang-prefill-1.log

# 检查GPU内存
```

**Q: Gateway连接不上?**
```bash
# 确认Prefill/Decode都在运行
curl http://127.0.0.1:30000/health

```

**Q: 请求超时?**
- 首次请求可能较慢（模型冷启动）
- 增加 `--max-time 120` 参数到curl

## Python测试脚本

如果你想用Python测试：

```python
import requests

GATEWAY_URL = "http://127.0.0.1:3000"

# 简单对话
resp = requests.post(f"{GATEWAY_URL}/v1/chat/completions", json={
    "model": "qwen2.5-0.5b-instruct",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 50
})
print(resp.json()["choices"][0]["message"]["content"])

# 并发请求
import concurrent.futures
def make_request(i):
    resp = requests.post(f"{GATEWAY_URL}/v1/chat/completions", json={
        "model": "qwen2.5-0.5b-instruct",
        "messages": [{"role": "user", "content": f"Request {i}"}],
        "max_tokens": 20
    })
    return resp.json()["choices"][0]["message"]["content"]

with concurrent.futures.ThreadPoolExecutor(max_workers=5) as executor:
    futures = [executor.submit(make_request, i) for i in range(1, 6)]
    for f in concurrent.futures.as_completed(futures):
        print(f.result())
```
