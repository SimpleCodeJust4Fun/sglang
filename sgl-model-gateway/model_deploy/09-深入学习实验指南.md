### 1.1 现有日志的不足

### 1.2 增强日志配置

#### 创建增强版启动脚本

```bash
# start-multi-pd-verbose.sh - 启动带详细日志的PD环境
#!/bin/bash

# 设置SGLang日志级别为DEBUG
export SGLANG_LOG_LEVEL=DEBUG

# 各组件日志输出到独立文件
# Prefill 1: 关注KV cache计算和bootstrap传输
setsid python3 -m sglang.launch_server \
    --model-path "$MODEL_PATH" \
    --port 30000 \
    --mem-fraction-static 0.15 \
    --tp 1 \
    --pd prefill \
    --disaggregation-bootstrap-port 9000 \
    --host 127.0.0.1 \
    --context-length 2048 \
    --log-level debug \
    > /tmp/sglang-prefill-1-verbose.log 2>&1 < /dev/null &

# Gateway: 关注路由决策和调度
./target/debug/sgl-model-gateway \
    --pd-disaggregation \
    --prefill http://127.0.0.1:30000 9000 \
    --prefill http://127.0.0.1:30001 9001 \
    --decode http://127.0.0.1:30010 \
    --decode http://127.0.0.1:30011 \
    --host 127.0.0.1 --port 3000 \
    --policy round_robin \
    --log-level debug \
    --log-file-path /tmp/gateway-verbose.log \
    > /tmp/gateway-verbose.log 2>&1 &
```

#### 日志文件清单

| 日志文件 | 组件 | 关键信息 |

| `/tmp/sglang-prefill-1.log` | Prefill-1 | KV cache计算、bootstrap传输 |

## 第二部分：系统化实验设计

### 实验1：验证PD分离架构的请求流程

**目标**: 确认请求的真实路径和响应返回机制

**步骤**:
```bash
# 1. 启动环境
bash start-multi-pd.sh

# 2. 启动Gateway（开启详细日志）
./target/debug/sgl-model-gateway \
    --pd-disaggregation \
    --prefill http://127.0.0.1:30000 9000 \
    --prefill http://127.0.0.1:30001 9001 \
    --decode http://127.0.0.1:30010 \
    --decode http://127.0.0.1:30011 \
    --host 127.0.0.1 --port 3000 \
    --policy round_robin \
    --log-level debug

# 3. 发送测试请求并计时
time curl -X POST http://127.0.0.1:3000/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model": "qwen2.5-0.5b-instruct", "messages": [{"role": "user", "content": "Hello"}], "max_tokens": 20}'

# 4. 同时监控所有日志
tail -f /tmp/gateway.log /tmp/sglang-prefill-1.log /tmp/sglang-decode-1.log
```

**观察要点**:
- [ ] Gateway是否同时发送请求到Prefill和Decode？
- [ ] Prefill是否计算KV cache并通过bootstrap传输？
- [ ] Decode是否直接返回响应给Gateway？
- [ ] 请求的时间线是否符合预期？

**预期结果**:
```
时间线:
T0: Gateway收到请求
T1: Gateway同时发送到Prefill和Decode
T2: Prefill计算KV cache
T3: Prefill通过bootstrap传输KV cache给Decode
T4: Decode基于KV cache生成token
T5: Decode直接返回响应给Gateway
T6: Gateway返回响应给客户端
```

### 实验2：对比三种调度策略

**目标**: 理解不同策略的工作原理和性能特征

#### 2.1 Round Robin（轮询）

**原理**: 按顺序轮流分配请求

**测试命令**:
```bash
# 启动Gateway
bash start-gateway-multi.sh round_robin

# 发送10个请求，观察分配模式
for i in {1..10}; do
    echo "=== Request $i ==="
    curl -s -X POST http://127.0.0.1:3000/v1/chat/completions \
        -H "Content-Type: application/json" \
        -d "{\"model\": \"qwen2.5-0.5b-instruct\", \"messages\": [{\"role\": \"user\", \"content\": \"Request $i\"}], \"max_tokens\": 10}" \
        | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"
    
    # 查看Gateway日志，确认路由到哪个Prefill-Decode对
    grep -E "(route|select)" /tmp/gateway.log | tail -1
done
```

**预期观察**:
- 请求均匀分配到 P1-D1, P2-D2, P1-D1, P2-D2...
- 每个请求的延迟相对稳定

#### 2.2 Cache Aware（缓存感知）

**原理**: 优先路由到有相关缓存的节点

**测试命令**:
```bash
# 启动Gateway
bash start-gateway-multi.sh cache_aware

# 发送相同前缀的请求，观察缓存命中
curl -s -X POST http://127.0.0.1:3000/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model": "qwen2.5-0.5b-instruct", "messages": [{"role": "user", "content": "Explain quantum physics"}], "max_tokens": 30}'

# 发送相似前缀的请求
curl -s -X POST http://127.0.0.1:3000/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model": "qwen2.5-0.5b-instruct", "messages": [{"role": "user", "content": "Explain quantum mechanics"}], "max_tokens": 30}'

# 查看缓存相关的Gateway日志
grep -E "(cache|prefix|match|tree)" /tmp/gateway.log | tail -20
```

**预期观察**:
- 相似前缀的请求可能被路由到同一个Prefill
- 第二次请求应该比第一次快（缓存命中）

#### 2.3 Random（随机）

**原理**: 随机选择节点

**测试命令**:
```bash
# 启动Gateway
bash start-gateway-multi.sh random

# 发送20个请求，统计分布
for i in {1..20}; do
    curl -s -X POST http://127.0.0.1:3000/v1/chat/completions \
        -H "Content-Type: application/json" \
        -d "{\"model\": \"qwen2.5-0.5b-instruct\", \"messages\": [{\"role\": \"user\", \"content\": \"Test $i\"}], \"max_tokens\": 10}" \
        > /dev/null &
done
wait

# 查看Gateway日志，统计路由分布
grep -E "(route|select)" /tmp/gateway.log | grep -oE "prefill-[12]" | sort | uniq -c
```

**预期观察**:
- 请求大致均匀分布，但不完全均匀
- 分布具有随机性

## 第三部分：自动化测试脚本

### 3.1 综合测试脚本

创建 `comprehensive-test.sh`:

```bash
#!/bin/bash
# 综合测试脚本 - 执行所有实验

set -e

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
BLUE='\033[0;34m'
CYAN='\033[0;36m'
NC='\033[0m'

RESULTS_DIR="/mnt/e/dev/sglang/sgl-model-gateway/model_deploy/experiment-results"
mkdir -p "$RESULTS_DIR"

echo -e "${BLUE}========================================${NC}"
echo -e "${BLUE}SGLang Model Gateway 综合实验${NC}"
echo -e "${BLUE}========================================${NC}"

# 实验1: 验证PD流程
run_experiment_1() {
    echo -e "\n${CYAN}[实验1] 验证PD分离架构的请求流程${NC}"
    
    local result_file="$RESULTS_DIR/exp1-pd-flow.md"
    echo "# 实验1: PD分离架构请求流程验证" > "$result_file"
    echo "" >> "$result_file"
    echo "## 测试时间: $(date)" >> "$result_file"
    echo "" >> "$result_file"
    
    # 发送请求并记录时间
    local start_time=$(date +%s%N)
    local response=$(curl -s -w "\n%{http_code} %{time_total}" \
        -X POST http://127.0.0.1:3000/v1/chat/completions \
        -H "Content-Type: application/json" \
        -d '{"model": "qwen2.5-0.5b-instruct", "messages": [{"role": "user", "content": "Hello"}], "max_tokens": 20}')
    local end_time=$(date +%s%N)
    
    local duration=$(( (end_time - start_time) / 1000000 ))
    
    echo "## 结果" >> "$result_file"
    echo "- 总耗时: ${duration}ms" >> "$result_file"
    echo "- 响应: $response" >> "$result_file"
    
    # 提取日志关键信息
    echo "" >> "$result_file"
    echo "## Gateway日志关键行" >> "$result_file"
    grep -E "(route|select|dispatch)" /tmp/gateway.log | tail -10 >> "$result_file" 2>/dev/null || echo "无相关日志" >> "$result_file"
    
    echo -e "${GREEN}实验1完成，结果保存到: $result_file${NC}"
}

# 实验2: 策略对比
run_experiment_2() {
    echo -e "\n${CYAN}[实验2] 对比三种调度策略${NC}"
    
    local result_file="$RESULTS_DIR/exp2-policy-comparison.md"
    echo "# 实验2: 调度策略对比" > "$result_file"
    
    for policy in round_robin cache_aware random; do
        echo -e "\n${YELLOW}测试策略: $policy${NC}"
        
        # 启动Gateway
        bash start-gateway-multi.sh $policy
        sleep 2
        
        echo "## 策略: $policy" >> "$result_file"
        echo "" >> "$result_file"
        
        # 发送10个请求
        local total_time=0
        for i in {1..10}; do
            local start=$(date +%s%N)
            curl -s -X POST http://127.0.0.1:3000/v1/chat/completions \
                -H "Content-Type: application/json" \
                -d "{\"model\": \"qwen2.5-0.5b-instruct\", \"messages\": [{\"role\": \"user\", \"content\": \"Test $i\"}], \"max_tokens\": 10}" \
                > /dev/null
            local end=$(date +%s%N)
            local req_time=$(( (end - start) / 1000000 ))
            total_time=$((total_time + req_time))
        done
        
        local avg_time=$((total_time / 10))
        echo "- 平均延迟: ${avg_time}ms" >> "$result_file"
        echo "- 总耗时: ${total_time}ms (10 requests)" >> "$result_file"
        echo "" >> "$result_file"
        
        # 停止Gateway
        killall sgl-model-gateway 2>/dev/null || true
        sleep 2
    done
    
    echo -e "${GREEN}实验2完成，结果保存到: $result_file${NC}"
}

# 运行所有实验
run_experiment_1
run_experiment_2

echo -e "\n${GREEN}========================================${NC}"
echo -e "${GREEN}所有实验完成！${NC}"
echo -e "${GREEN}========================================${NC}"
echo -e "结果目录: $RESULTS_DIR"
ls -la "$RESULTS_DIR"
```
