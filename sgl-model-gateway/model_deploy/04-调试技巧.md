## 🐛 常见错误与解决方案

### 错误 1: CUDA Out of Memory (OOM)

**完整错误信息**:
```
RuntimeError: CUDA out of memory. Tried to allocate XXX MiB
```

**原因分析**:
1. 模型 + KV缓存 + 其他开销超过显存
2. `--mem-fraction-static` 设置过高
3. 批处理大小过大

**解决方案**:

**方案 A: 降低显存占用**
```bash
# 编辑 start_sglang.sh
--mem-fraction-static 0.6  # 从 0.7 降低到 0.6
--max-running-requests 2   # 限制并发请求数
```

**方案 B: 使用更激进的量化**
```bash
# 如果有 INT4 版本，优先使用
# 或考虑 CPU offload
--cpu-offload-gb 2  # 将 2GB 参数卸载到 CPU
```

**方案 C: 检查显存占用**
```bash
# 启动前检查
nvidia-smi

# 关闭其他占用显存的程序
pkill -f "其他程序名"
```

### 错误 2: Port Already in Use

**完整错误信息**:
```
OSError: [Errno 98] Address already in use
```

**解决方案**:

**方案 A: 更换端口**
```bash
# 编辑 start_sglang.sh，修改端口
--port 30001  # 使用其他端口
```

**方案 B: 释放端口**
```bash
# 查找占用进程
lsof -i :30000
# 或
netstat -tuln | grep 30000

# 终止进程
kill -9 <PID>
```

### 错误 3: Model Loading Failed

**完整错误信息**:

```bash
export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH
```

## 🔬 性能问题诊断

### 1. 吞吐量低

**症状**: tokens/s 远低于预期

**诊断步骤**:

```bash
# 1. 检查 GPU 利用率
nvidia-smi

# 2. 查看日志中的 batch size
# 查找 "Prefill batch" 和 "Decode step"

# 3. 检查是否启用了 CUDA Graph
# 如果日志中有 "disable_cuda_graph=True"，性能会降低
```

**优化建议**:

```bash
# 启用 CUDA Graph（需要足够显存）
# 移除 start_sglang.sh 中的 --disable-cuda-graph

# 增加并发请求数
--max-running-requests 8

# 调整 chunked prefill
--chunked-prefill-size 4096
```

### 2. 延迟高

**症状**: 首 token 延迟 > 1秒

**诊断步骤**:

```python
import time
import requests

url = "http://localhost:30000/v1/chat/completions"

# 测试首 token 延迟
start = time.time()
response = requests.post(url, json={
    "model": "default",
    "messages": [{"role": "user", "content": "hi"}],
    "stream": True
})

first_token_time = None
for line in response.iter_lines():
    if first_token_time is None and line:
        first_token_time = time.time() - start
        print(f"首 token 延迟: {first_token_time:.3f}s")
        break
```

**优化建议**:

```bash
# 减少并发请求数
--max-running-requests 2

# 降低 prefill 批大小
--chunked-prefill-size 1024

# 确保没有其他程序占用 GPU
```

### 3. 显存使用不合理

**症状**: 显存使用率过高或过低

**诊断工具**:

```python
import requests

# 获取服务器信息
info = requests.get("http://localhost:30000/get_server_info").json()
print(f"显存使用: {info.get('memory_used', 'N/A')} GB")
print(f"显存总量: {info.get('memory_total', 'N/A')} GB")
print(f"缓存使用: {info.get('cache_usage', 'N/A')}%")
```

**调整策略**:

```bash
# 显存使用过低 - 可以增加缓存
--mem-fraction-static 0.8

# 显存使用过高 - 需要降低
--mem-fraction-static 0.6

# 动态调整 KV 缓存大小
--max-total-tokens 100000  # 限制总 token 数
```

### 2. 压力测试脚本

```python
import time
import asyncio
import aiohttp
import statistics

async def single_request(session, id):
    start = time.time()
    try:
        async with session.post(
            "http://localhost:30000/v1/chat/completions",
            json={
                "model": "default",
                "messages": [{"role": "user", "content": f"测试 {id}"}],
                "max_tokens": 50
            },
            timeout=aiohttp.ClientTimeout(total=60)
        ) as response:
            await response.json()
            return time.time() - start, True
    except Exception as e:
        return time.time() - start, False

async def stress_test(total_requests=100, concurrency=10):
    print(f"开始压力测试: {total_requests} 请求, 并发 {concurrency}")
    
    start_time = time.time()
    latencies = []
    failures = 0
    
    async with aiohttp.ClientSession() as session:
        for batch_start in range(0, total_requests, concurrency):
            batch_end = min(batch_start + concurrency, total_requests)
            tasks = [
                single_request(session, i)
                for i in range(batch_start, batch_end)
            ]
            results = await asyncio.gather(*tasks)
            
            for latency, success in results:
                if success:
                    latencies.append(latency)
                else:
                    failures += 1
            
            # 显示进度
            progress = (batch_end / total_requests) * 100
            print(f"进度: {progress:.0f}% ({batch_end}/{total_requests})")
    
    total_time = time.time() - start_time
    
    # 统计结果
    print(f"\n{'='*50}")
    print(f"总请求数: {total_requests}")
    print(f"成功: {len(latencies)}, 失败: {failures}")
    print(f"总耗时: {total_time:.2f}s")
    print(f"吞吐量: {total_requests/total_time:.2f} req/s")
    
    if latencies:
        print(f"\n延迟统计:")
        print(f"  平均: {statistics.mean(latencies):.3f}s")
        print(f"  中位数: {statistics.median(latencies):.3f}s")
        print(f"  P95: {sorted(latencies)[int(len(latencies)*0.95)]:.3f}s")
        print(f"  P99: {sorted(latencies)[int(len(latencies)*0.99)]:.3f}s")
        print(f"  最小: {min(latencies):.3f}s")
        print(f"  最大: {max(latencies):.3f}s")

# 运行测试
asyncio.run(stress_test(total_requests=50, concurrency=5))
```

### 2. 请求延迟分析

```python
import time
import requests
import matplotlib.pyplot as plt

def measure_latency(num_requests=50):
    latencies = []
    
    for i in range(num_requests):
        start = time.time()
        response = requests.post(
            "http://localhost:30000/v1/chat/completions",
            json={
                "model": "default",
                "messages": [{"role": "user", "content": f"请求 {i}"}],
                "max_tokens": 50
            }
        )
        latency = time.time() - start
        latencies.append(latency)
        print(f"请求 {i}: {latency:.3f}s")
    
    # 绘制延迟分布图
    plt.figure(figsize=(10, 6))
    plt.plot(latencies)
    plt.xlabel('请求序号')
    plt.ylabel('延迟 (秒)')
    plt.title('请求延迟分布')
    plt.grid(True)
    plt.savefig('latency_distribution.png')
    print(f"延迟图表已保存到 latency_distribution.png")
    
    return latencies

# 运行分析
latencies = measure_latency(50)
```
