# PD Disaggregation 完整测试指南

本文档提供在 WSL2 + RTX 4070 Ti SUPER (16GB) 环境下测试 SGLang Model Gateway PD 分离架构的完整步骤。

- ✅ WSL2 Ubuntu 24.04
- ✅ NVIDIA RTX 4070 Ti SUPER (16GB VRAM)
- ✅ Python 3.12 + 虚拟环境 (`~/qwen_env`)
- ✅ SGLang 已安装
- ✅ Qwen2.5-7B-Instruct-AWQ 模型已下载
- ✅ Rust 工具链 (用于编译 Gateway)

### 模型路径
```
/home/tyliu/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ
```

## 🚀 快速开始

### 方案二：手动分步执行

#### 步骤 1: 激活环境并检查 GPU

```bash
# 激活 Python 虚拟环境
source ~/qwen_env/bin/activate

# 检查 GPU
nvidia-smi
```

#### 步骤 2: 启动 Prefill Server

```bash
source ~/qwen_env/bin/activate

python3 -m sglang.launch_server \
    --model-path /home/tyliu/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ \
    --port 30000 \
    --mem-fraction-static 0.3 \
    --tp 1 \
    --pd prefill \
    --disaggregation-bootstrap-port 9000 \
    --host 127.0.0.1
```

**参数说明：**
- `--mem-fraction-static 0.3`: 使用 30% 显存（约 4.8GB），为 Decode Server 留空间
- `--pd prefill`: 指定为 Prefill 角色
- `--disaggregation-bootstrap-port 9000`: KV Cache 传输端口

**等待启动完成**（约 30-60 秒），看到以下日志表示成功：
```
The server is fired up and ready to roll!
```

验证健康状态：
```bash
curl http://127.0.0.1:30000/health
```

#### 步骤 3: 启动 Decode Server

打开**新的终端**：

```bash
source ~/qwen_env/bin/activate

python3 -m sglang.launch_server \
    --model-path /home/tyliu/.cache/modelscope/hub/models/qwen/Qwen2___5-7B-Instruct-AWQ \
    --port 30001 \
    --mem-fraction-static 0.3 \
    --tp 1 \
    --pd decode \
    --host 127.0.0.1
```

**参数说明：**
- `--pd decode`: 指定为 Decode 角色
- 使用相同的 `--mem-fraction-static 0.3`

等待启动完成后验证：
```bash
curl http://127.0.0.1:30001/health
```

#### 步骤 4: 检查显存使用

```bash
nvidia-smi
```

你应该看到两个 python 进程，各占用约 4-5GB 显存。

#### 步骤 5: 编译 Gateway

```bash
cd /mnt/e/dev/sglang/sgl-model-gateway

# 如果还未编译
cargo build

# 编译完成后检查二进制
ls -lh target/debug/sgl-model-gateway
```

#### 步骤 6: 启动 Gateway

```bash
cd /mnt/e/dev/sglang/sgl-model-gateway

./target/debug/sgl-model-gateway \
    --pd-disaggregation \
    --prefill http://127.0.0.1:30000 9000 \
    --decode http://127.0.0.1:30001 \
    --host 127.0.0.1 \
    --port 3000 \
    --policy round_robin \
    --log-level debug
```

**参数说明：**
- `--pd-disaggregation`: 启用 PD 分离模式
- `--prefill http://127.0.0.1:30000 9000`: Prefill Server URL + Bootstrap Port
- `--decode http://127.0.0.1:30001`: Decode Server URL
- `--policy round_robin`: 使用轮询策略

看到以下日志表示成功：
```
SGLang Router starting...
Mode: PD Disaggregated
Prefill nodes: [("http://127.0.0.1:30000", Some(9000))]
Decode nodes: ["http://127.0.0.1:30001"]
```

## 🔍 观察要点

### 1. Prefill Server 日志

查看日志：
```bash
tail -f /tmp/sglang-prefill.log
```

**关键观察：**
- 收到包含 `bootstrap_host`, `bootstrap_port`, `bootstrap_room` 的请求
- 完成 prefill 计算
- 通过 bootstrap 端口传输 KV Cache

### 2. Decode Server 日志

查看日志：
```bash
tail -f /tmp/sglang-decode.log
```

**关键观察：**
- 等待并接收来自 Prefill 的 KV Cache
- 开始 decode 过程
- 生成 token 并返回响应

### 3. Gateway 日志

在 Gateway 运行终端观察：

**关键观察：**
- Worker 注册和 health check
- 请求路由决策
- bootstrap 信息注入
- 错误处理和重试

### 4. GPU 使用情况

```bash
# 实时监控
watch -n 1 nvidia-smi

# 查看进程详情
nvidia-smi pmon -i 0
```

**预期行为：**
- 请求处理时，两个进程的 GPU 利用率都会上升
- Prefill 阶段：Prefill Server GPU 利用率高
- Decode 阶段：Decode Server GPU 利用率高

## ❌ 故障排查

### 问题 1: Prefill/Decode Server 启动失败

**症状：** 启动后很快退出或报错

**解决方法：**
```bash
# 查看详细日志
tail -100 /tmp/sglang-prefill.log
tail -100 /tmp/sglang-decode.log

# 常见原因：
# 1. 显存不足 - 降低 --mem-fraction-static (如 0.25)
# 2. 模型路径错误 - 确认路径存在
# 3. 端口冲突 - 更改端口号
```

### 问题 2: Gateway 无法连接 Worker

**症状：** Gateway 日志显示 health check 失败

**解决方法：**
```bash
# 检查 Worker 是否运行
curl http://127.0.0.1:30000/health
curl http://127.0.0.1:30001/health

# 检查网络连通性
curl -v http://127.0.0.1:30000/health

# 确认防火墙规则（WSL2 通常不需要）
```

### 问题 3: 请求超时

**症状：** curl 请求长时间无响应

**解决方法：**
```bash
# 检查 Gateway 是否在处理请求
# 查看 Gateway 日志输出

# 增加超时时间
curl --max-time 120 -X POST http://127.0.0.1:3000/v1/chat/completions ...

# 检查 GPU 是否过载
nvidia-smi
```

### 问题 4: OOM (Out of Memory)

**症状：** 进程被 kill 或报显存不足

**解决方法：**
```bash
# 降低显存占用
--mem-fraction-static 0.25  # 从 0.3 降到 0.25

# 或限制上下文长度
--context-length 8192  # 从 32768 降到 8192

# 清理其他 GPU 进程
pkill -f "sglang"
nvidia-smi
```

## 💡 提示

1. **首次启动慢是正常的**：模型加载需要时间，后续请求会快很多
2. **日志是关键**：遇到问题首先查看三个组件的日志
3. **显存管理**：16GB 显存跑两个 7B 实例比较紧张，注意监控
4. **网络配置**：WSL2 中使用 `127.0.0.1` 而不是 `0.0.0.0`
5. **进程管理**：使用 `cleanup-pd-test.sh` 确保干净退出
